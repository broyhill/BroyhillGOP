FILE 1: base_agent.py (709 lines, ~26 KB)
========================================

The foundation class for all ecosystem agents. Provides:

ARCHITECTURE:
- Async/await with asyncio and httpx
- Single inheritance base class
- Abstract methods for subclass implementation
- Signal handling (SIGTERM, SIGINT)

CORE MONITORING LOOPS (4 parallel async loops):
1. health_check() - Monitors ecosystem health every 60 seconds
2. audit_data_quality() - Data quality audits every 300 seconds
3. check_performance() - Performance metrics every 120 seconds
4. check_domain_rules() - Domain rules every 180 seconds

HEARTBEAT SYSTEM:
- _heartbeat_loop() - Writes heartbeats every 30 seconds
- _write_heartbeat() - Records to agent_heartbeats table
- Tracks: status, uptime, API calls, costs

EVENT EMISSION:
- _emit_event() - Writes to agent_events table
- Includes: event_type, severity, timestamp, details
- Severity levels: info, warning, critical, emergency

RULE EVALUATION ENGINE:
- _evaluate_rules() - Evaluates all active rules against metrics
- _refresh_rules_cache() - Hot-reload every 5 minutes
- _load_rules_from_database() - Loads from agent_rules table
- _evaluate_single_rule() - Single rule evaluation
- _compare_values() - Operator support:
  * Comparison: gt, lt, gte, lte, eq, neq
  * Pattern: contains, regex
  * Range: between (inclusive)
- _is_on_cooldown() - Prevents alert storms

METRIC RECORDING:
- _record_metric() - Records to agent_metrics table
- Supports: gauge, counter, histogram types
- Includes tags and timestamps

COST TRACKING:
- _track_api_call() - Tracks API calls and costs
- Accumulates api_call_count and total_api_cost
- Per-call cost tracking

BRAIN INTEGRATION:
- _brain_directive_loop() - Checks every 60 seconds
- _check_brain_directives() - Loads approved directives
- _execute_brain_directive() - Executes directives
- Directive types: adjust_threshold, toggle_control, enable/disable_rule
- _update_directive_status() - Marks executed/failed

HTTP REQUEST HANDLING:
- _http_request() - With retry logic (exponential backoff)
- Configurable retries (default: 3) and delays
- Connection pooling with httpx
- Automatic timeout handling

INITIALIZATION & SHUTDOWN:
- initialize() - Sets up HTTP client and registry
- shutdown() - Graceful cleanup
  * Cancels all tasks
  * Closes HTTP client
  * Writes final heartbeat
- run() - Main entry point, starts all loops

ABSTRACT METHODS (Override in subclasses):
- _perform_health_check() -> Dict[str, Any]
- _perform_data_quality_audit() -> Dict[str, Any]
- _perform_performance_check() -> Dict[str, Any]
- _check_domain_specific_rules() -> None

DEPENDENCIES:
- asyncio (stdlib)
- httpx (async HTTP client)
- json (stdlib)
- logging (stdlib)
- signal (stdlib)
- datetime (stdlib)
- abc (stdlib)

===========================================

Standalone rule evaluation engine with hot-reload support.

RULE EVALUATION:

- evaluate_all() - Evaluate all rules against metrics

RULE MANAGEMENT (CRUD):

- update_rule() - Modify existing rule
- delete_rule() - Remove rule
- toggle_rule() - Enable/disable without deletion

HOT-RELOAD:

- Force reload capability
- Stale cache detection
- No restarts required

INTEGRATION WITH BRAIN:

STATISTICS:

DATA STRUCTURES:
- rules: Dict[str, Dict] - ID to rule mapping

- reload_interval: 300 seconds

FILE 6: notifier.py (308 lines, ~11 KB)
========================================

Multi-channel notification system with rate limiting.

NOTIFICATION CHANNELS:
1. Slack - With color coding by severity
   * Emergency: Red (#cc0000)
   * Critical: Orange (#ff6600)
   * Warning: Yellow (#ffaa00)
   * Info: Blue (#0099cc)

2. Email - SMTP-based (configured via env)
   * Supports plaintext and HTML

3. SMS - Twilio integration
   * For urgent alerts

4. Dashboard - Supabase insert
   * Stored as agent_notifications
   * Can be marked read

5. Webhooks - Arbitrary POST endpoints
   * Custom integration support

RATE LIMITING:
- Per-channel rate limiting (default: 5 minutes)
- Prevents notification storms
- Separate cooldowns per channel

DEDUPLICATION:
- _hash_notification() - MD5 hash of event+message
- _is_duplicate() - Checks against history
- Prevents sending same alert twice in window

ESCALATION:
- _check_escalation() - 30-minute timer on critical alerts
- Escalates to emergency if not acknowledged
- Automatic escalation notification

MESSAGE FORMATTING:
- _format_message() - Severity + message + details
- Severity-specific coloring
- Timestamp inclusion
- Detail serialization

ERROR HANDLING:
- Try-catch on all channels
- Continues if one channel fails
- Logs failures for debugging

DEPENDENCIES:
- asyncio
- httpx
- hashlib
- datetime
- logging

Cost tracking, budgeting, and optimization.

  * ecosystem_id

  * cost in dollars
  * record_count for per-record calculations
  * Inserts to agent_cost_tracking table

COST SUMMARIES:

  * total_cost
  * total_records
  * average_cost_per_record
  * breakdown by api_type

BUDGET VARIANCE:
- calculate_variance() - Budget vs actual analysis
  * Per-ecosystem breakdown
  * Variance and variance_percent
  * Status: over_budget / under_budget
  * Flags ecosystems over threshold
  * generate_variance_report() - Human-readable text report

OPTIMIZATION:
- optimize_budget_allocation() - Linear programming
  * Given total budget, allocate to ecosystems
  * Weighted by historical ROI
  * Equal allocation fallback if no ROI data
  * Returns Dict[ecosystem_id, allocation]

DATA AGGREGATION:
- _group_costs_by_type() - Summarize by API type
- Loads from agent_cost_tracking table
- Groups by ecosystem and api_type

IN-MEMORY CACHING:
- ecosystem_costs: Dict[str, Dict[str, Any]]
- Reduces database queries
- Updated on track_api_call()
