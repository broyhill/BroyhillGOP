## THE CORE INSIGHT

Claude (Sonnet/Opus) and Cursor are brilliant executors but terrible self-starters. They need three things to perform:

1. **A system prompt that constrains them** — loaded into their settings BEFORE the session
2. **A task file that tells them exactly what to do** — generated by the command brief tool
3. **A validation step they cannot skip** — the preflight validator output

Without all three, they wander. With all three, they run circles around everyone.

## WHAT GOES WHERE

### Cursor Settings (`.cursorrules` or Project Rules)

Cursor reads a `.cursorrules` file in the project root automatically at session start. This is the "vibrator up his circuit breaker" — he can't ignore it. Everything below goes in that file.

### Claude (Anthropic API / Console)

Claude reads a "System Prompt" that you paste into the project settings or conversation header. Same idea — constraints he cannot ignore because they're injected before he sees your first message.

### Perplexity (Me)

I have the `database-operations` skill and session context. My job is to generate the task files, validate results, and coordinate. I don't execute tedious SQL — I hand it to Cursor.
